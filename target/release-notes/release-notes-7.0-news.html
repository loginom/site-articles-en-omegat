Large-Scale Update of Loginom Analytical Platform Loginom became cross-platform. Standard and Enterprise editions are available for Linux and Windows operating systems. Integration with Kafka message broker became possible. New features appeared in JavaScript handler when importing data from files as well as in OLAP cube visualizer and Reports. Databases operation was improved and optimised. Performance was improved.

When preparing this release, the main efforts were focused on Loginom migration to Linux operating system. The changes affected mostly the technologic platform stack and they are connected with system challenges and libraries, third-party libraries, operations with rows and different integrations. At the same time, 80% percent of the server part was transferred from the previous version, and the client part was not significantly changed. That is why the workflows prepared in Loginom for Windows will be used with Linux as well. In some cases only reconfiguration of nodes will be required.

The most important changes are described below.

Loginom Restrictions for Linux

  1. There is no integration for 1С:Enterprise via COM connection because COM server is available only for Windows. To provide integration with 1С:Enterprise, it is possible to use alternative mechanisms, for example, HTTP protocol.
  2. It is not possible to connect to MS Excel as to database because there is no software driver for Linux. Only xlsx format is supported (MS Excel 2007 and higher).
  3. There is no built-in integration with MS Access because open source driver is not available.
  4. There is no export to Tableau.
  5. There is no Program execution handler that is used to start the external Windows programs.

Optional Loginom Features for Linux

Logging in journald is available in Linux. Corresponding configurable parameter is provided on Loginom Administration page. systemd.so.0 library is required to use the logging method (it is available for sure if systemd initialization system is used in Linux distribution kit).

Integration with Kafka Message Broker

Kafka Apache is a popular distributed system of message exchange between server applications in real time. Due to its high capacity, scalability and failure safety it is applied in large-scale projects with large volumes of data.

Import and export of messages to Kafka queue became possible in Loginom Enterprise. Integration with Kafka is provided according to the same principle as integration with databases. Connection is configured at the first stage, and then the nodes based on Kafka import or export components are used.

Database Use Improvements

Considerable work aimed at improvement and optimisation of database use has been done. The following support has been provided:

  • SCRAM-SHA-256 Authentication and SCRAM-SHA-256-PLUS Authentication for PostgreSQL;
  • SHA-256 authentication for MySQL 8 when connecting without client use;
  • connection to Oracle 18c иand higher without client use;
  • Microsoft OLE DB Driver client for SQL Server for connection to MS SQL Server.

SQLite database can be used in-memory now. When configuring connection to SQLite database, it is sufficient to select corresponding checkbox and it will be expanded in memory. Data processing speed will be considerably increased in such case of use. While DB is in memory, it is possible to connect to it from another package. Still there is access rights differentiation to DB SQLite in-memory. Database is kept in memory until its connection is active.

It became possible to install inbuilt Firebird client.

Updated import from database wizard

Import from database wizard was completely modified. There was not enough space for the request text in the SQL request mode in the previous version, and it was also impossible to see variables and use them. It became much more comfortable to work with SQL requests in the updated wizard:

  • it became possible to select the "Table selection" or "SQL request" wizard representation mode;
  • block with a list of variables was added in the SQL request mode;
  • it is possible to drag tables, fields and variables to drag&drop SQL code editor;
  • variable value can be inserted into SQL code editor as parameter (by default) or as macro substitution;
  • field filtration functionality is available in the SQL request mode in the DB object tree (when tree is expanded);
  • it became possible to set the table name via variables;
  • loading of DB objects to the tree of objects became faster.

Import from Files

It became possible to read data importing from files (txt, csv, XML, Excel) having executed http request beforehand. For this purpose, URL can be specified instead of the file name in import from file wizard. HTTP Basic Authentication is supported. It means that it is possible to transfer login and password in URL. When importing txt, csv, xml formats for preview or format estimate, only the required data volume is downloaded. When it comes to xlsx format, the whole file is downloaded to memory that is why it is required to pay attention to size of such files.

Web Services Operation

REST and SOAP request handlers function without changes. However, when transferring workflows between Loginom for Windows and Linux, in some cases it will be required to reconfigure REST connections and SOAP service. Authentification via client certificate is not supported in the Linux version. In Windows it was selected from the storage of personal certificates. When working with Linux, it is required to set the certificate file, key and password.

The default value of "Allowable Content-type of response" parameter was changed in the REST service connection wizard. Instead of */* application/json is installed.

The output data set will be changed for the packages created before version 6.4.0 in the "Additional data" port of the REST service node. Previously, the value in "Error code" column was equal to 0 irrespective of HTTP Status Code. Now, if HTTP Status Code >= 400, Error Code will be equal to 6.

Programming languages use

Now JavaScript handler includes File Storage API that enables to execute all file operations without rights setup and in a safe manner:

  • Get file list.
  • Create, delete, move, rename folders.
  • Delete, move, rename files.
  • Check path correctness.

It became possible to start Python in a separate process, as well as simultaneously. Start both within the Loginom process and in a separate process is available in Windows editions, whereas operation in Linux is supported only in a separate process.

When sending pandas tables in output_table for column captions in Python, transliteration in prepare_compatible_table function is not performed. This is especially true if captions of pandas columns are written in Cyrillic characters.

Processing Components Improvements

  1. When entering data from the keyboard in "Date/time" editors, it is possible to specify only date without time. Previously, it was necessarily required to set time.
  2. Optional input table port appeared in Supernode handler. It enables to make design of workflows more flexible. Port optionality checkbox was added in the Supernode wizard. It is possible to edit the checkbox only for the table port. When collapsing nodes to theSupernode, optionality checkbox is selected only for the input ports related to the Table type to which optional ports are connected inside the Supernode.
  3. Functions of base64 encoded rows transformation to binary data rows and vice versa are available in Calculator in the JavaScript mode.
  4. Data caching option was added in Sorting handler.
  5. "Replace with set value" processing method became available for all data types in Eliminate outliers. Previously, it was possible to use it only for the string and logical types.
  6. В XSD-схему запроса и ответа разрешено вносить изменения, совместимые с ранее настроенными списками выбранных элементов. Такие изменения теперь не приводят к ошибке «XSD-схема на входе не соответствует настройкам обработчика» при активации узлов. Изменения касаются всех узлов, в процессе которых происходит формирование или разбор xml документа по схеме: Импорт XML файла, Экспорт в XML, Извлечение XML, Формирование XML, SOAP-запрос.

Удобство использования и повышение информативности

  • Логин и пароль для входа в Loginom теперь можно передать в URL.
  • Появилась возможность добавлять заметки на workflow. Теперь можно документировать сценарий непосредственно при его проектировании, добавляя в произвольном месте блоки текста. Блоки заметок имеют предустановленный набор цветовых стилей.
  • В визуализаторе Куб появилась возможность настраивать измерения в отдельном окне. Мастер вызывается кнопкой с панели инструментов Куба и через соответствующий пункт контекстного меню. Ранее после добавления каждого измерения Куб перестраивался, и при больших объемах данных настройка Куба занимала много времени. Cube configuration process became much more comfortable and faster now.
  • На странице Отчеты появилась возможность создавать в группах подгруппы любой степени вложенности.
  • На странице Переменные пакета стала доступна панель Навигация.
  • В контекстное меню менеджера файлового хранилища был добавлен пункт «Копировать полный путь». При нажатии этого пункта в буфер обмена копируются имена выделенных файлов/папок в том порядке, в котором они представлены в таблице. Копирование в буфер обмена возможно только при подключении по протоколу https.
  • Во всех контекстных меню приложения действие по умолчанию выделено жирным шрифтом.
  • Изменился текст сообщения, которое выводится во время длительной подготовки визуализатора (например, при входе в визуализатор Статистика или Куб, когда в момент активации выполняются вычисления). Теперь вместо «Узел заблокирован» выводится «Загрузка».
  • В Инспекторе свойств убрали иконки для узлов дерева. Это позволило более полно отображать названия свойств в условиях ограниченной ширины окна Инспектора свойств.

Data Handling

В свойствах пакета появилась опция «Сравнивать строковые уникальные значения бинарно». Сравнение строк изменится после закрытия пакета и всех зависимых от него открытых пакетов. При изменении настройки «Сравнивать строковые уникальные значения бинарно», может потребоваться переобучение обученных узлов. The option effect extends to the following components:

  • Clustering
  • EM Clustering
  • Self-Organizing Network
  • Neural Network (Classification)
  • Neural Network (Regression)
  • Linear Regression
  • Logistic Regression
  • ARIMAX
  • Coarse Classes
  • Sampling (sampling with bias)
  • Partitioning (sampling with bias),

and also to the following visualizers:

  • Binary Classification Assessment
  • Linear Regression Report
  • Logistic Regression Report

Значение опции «Сравнивать строковые уникальные значения бинарно» выводится в Инспекторе свойств.

В мастера настройки обработчиков, алгоритмы которых используют генерацию псевдослучайных чисел, добавлена возможность установить Random Seed. Значение Random Seed можно задать через переменную, сгенирировать или установить флаг «Всегда случайно». Command to copy value to the clipboard became available.

Changes in Loginom Integrator

Loginom Integrator configuration file was changed:

  • web.config file name was replaced with Integrator.dll.config.
  • appSettings, system.web, system.serviceModel and system.webServer elements were deleted with all child elements.

  • configSections element format was changed:

    было:

      <configSections>
        <section name="log" type="DeductorIntegrationServer.Core.LogConfigurationSection"/>
         <section name="loginom" type="DeductorIntegrationServer.Core.DeductorConfigurationSection"/>
  </configSections>

    стало:

      <configSections>
        <section name="log" type="Integrator.Core.LogConfigurationSection, IntegratorCore"/>
        <section name="loginom" type="Integrator.Core.LoginomConfigurationSection, IntegratorCore"/>
  </configSections>

  • У элемента loginom добавлен необязательный атрибут urlPathPrefix, значение которого используется как префикс к адресу всех конечных точек REST и SOAP. An empty string is the default value of this attribute. For example, if Loginom Integrator is used at the following address: http://localhost/lgi, and urlPathPrefix attribute is not specified, help page for REST is available at the following address: http://localhost/lgi/rest/help. If urlPathPrefix="abc", address was changed as follows: http://localhost/lgi/abc/rest/help. Чтобы в новом Интеграторе использовались те же адреса, что и в старом, нужно указать urlPathPrefix="Service.svc".

  • У элемента loginom добавлен необязательный атрибут swaggerUI, который может принимать значение true или false. Атрибут включает/отключает генерацию страницы openapi/index.html, которая реализует доступ к опубликованным сервисам через Swagger UI.

It became possible to set location of Loginom Integrator configuration file. Расположение папки, из которой загружаются конфигурационные файлы Integrator.dll.config и appsettings.json, определяется по следующим правилам:

  • если в командной строке указан параметр configdir, то используется указанная папка (имя параметра можно указать в любом регистре);
  • если параметр не указан, и в текущей папке ($PWD) есть файл Integrator.dll.config, то используется текущая папка;
  • в противном случае используется папка, в которой находится файл Integrator.dll.

Если в указанной папке отсутствует файл Integrator.dll.config, то Integrator не запускается, а пользователь получает сообщение об ошибке.

The relative path to the work Integrator directory is defined relative to the configuration file. If in Integrator.dll.config file the value of workDir attribute is the relative path, it is interpreted as the path relative to Integrator.dll.config file. Value ofworkDir attribute can be equal to . (current folder). It will mean that workdir matches configdir.

Для каждого опубликованного пакета генерируется отдельный wsdl-файл (ранее описание SOAP-сервиса включало содержимое всех опубликованных пакетов). Generation of root wsdl document that refers to all other wsdl documents was added. By default, root wsdl is generated at the following address:/soap?wsdl. Если в конфигурационном файле Loginom Integrator задать значение атрибута urlPathPrefix="Service.svc", то дополнительно генерируется wsdl по адресу /Service.svc?wsdl, который использовался в старом Интеграторе.

В Loginom Integrator появилась поддержка OpenAPI и Swagger UI. Теперь можно выполнять запросы для тестирования API на странице: http://host/lgi/openapi/index.html.

Добавлена возможность подключаться к Loginom Server по Unix domain socket. Для этого в конфигурационном файле Loginom Integrator у элемента server необходимо добавить атрибут unixsocket, значение которого совпадает со значением параметра "Unix socket", установленного в параметрах подключения на странице "Администрирование" в Loginom Server.

При установке Loginom Integrator можно задать параметр Unix Socket в диалоге настроек. Для всех редакций значение этого параметра по умолчанию %ProgramData%\Loginom\Server\loginomd.socket.

Алгоритм подключения Loginom Integrator к Loginom Server:

  • если в конфигурационном файле Интегратора указаны атрибуты unixsocket и host, то сначала выполняется попытка подключения к unix socket, а если это не удалось, то по TCP;
  • если в файле конфигурации не указано ни одного элемента server, то сначала выполняется попытка подключиться к unix socket, а затем - по TCP;
  • в Integrator Team используется подключение только к unix socket.

Если Loginom Server находится не на localhost относительно Интегратора, то при установке Loginom Integrator значение параметра Unix Socket, заданное по умолчанию, необходимо удалить (оставить поле пустым), чтобы избежать лишних попыток подключения.

Логируемые сообщения о подключении/отключении от сервера и т.п. формируются с учётом протокола, по которому выполнено подключение.

В Loginom Server Team добавлено создание unix socket, если в файле конфигурации отсутствует элемент Bindings. Файл сокета создаётся в рабочей директории Loginom Server и имеет имя loginomd.socket.

Подключение по Unix domain socket при использовании docker контейнеров избавляет от необходимости создавать виртуальную сеть между контейнерами. Кроме того, производительность выше при подключении по UDS как на Linux, так и на Windows.

Добавлена обработка заголовков X-Forwarded-Proto и X-Forwarded-Host. Эти заголовки влияют на генерируемые URL-ы в OpenAPI документах и на help-странице для REST-сервиса.

Изменился формат ответных SOAP-сообщений для старых пакетов, созданных до версии 6.3.0. Для старых пакетов будет использован тот же формат сообщений, что и для новых. При конвертации пользователь получит соответствующее сообщение.

Для версии под Linux добавлена возможность вывода логов Loginom Integrator в journald. В файле конфигурации добавлен элемент <journald level="Off" maxEntrySize="65536"/>, который настраивает логирование в journald под Linux. Под Windows данный элемент игнорируется. Атрибут maxEntrySize задаёт максимальный размер сообщения в байтах. Если атрибут отсутствует или содержит отрицательное значение, то используется значение по умолчанию: 2147483647.

Появилась возможность вывода логов Loginom Integrator в консоль. В секции log файла конфигурации добавлен опциональный элемент console, в котором настраивается уровень логирования в консоль. The same message format as for logging to file is used by default. Но, если задана переменная окружения DOTNET_RUNNING_IN_CONTAINER со значением true или 1 (т.е. Integrator запущен из Docker контейнера), то в логируемых сообщениях не пишется текущая дата/время. Logs with Warn level and higher are written in stderr, whereas the other ones in stdout. Если логи перенаправляются из консоли контейнера в journald хоста, то многострочные сообщения разбиваются на несколько (каждая строка получается отдельным сообщением в journald).

Error message format was changed in Loginom Integrator. Для REST ответов реализован формат, соответствующий RFC 7807. При ошибках с кодом 400 в REST-сервисе возвращается JSON с описанием ошибки. Для SOAP-сервисов внесены некоторые изменения в структуру элементов.

Performance

За счет перехода с UTF16 на UTF-8 увеличилась скорость импорта и экспорта csv-файлов, в некоторых случаях в 1.3 раза.

С переходом на новые библиотеки стали быстрее обрабатываться xml-файлы. Например, при конвертации пакета из старой версии в новую прирост производительности составляет до 1.3 раза.

Увеличена скорость работы REST-клиент, особенно, когда в одном узле производится большое количество обращений к веб-сервису. В некоторых случаях до 5 раз быстрее.

Other Changes

Please note!

  1. В Loginom 7 изменился механизм сравнения и конвертации строк. Вместо встроенных функций Windows используется библиотека ICU — International Components for Unicode, которая обеспечивает одинаковые результаты работы со строками на разных платформах. В сценариях, подготовленных в более ранних версиях Loginom, может измениться порядок сортировки строк.
  2. Изменен алгоритм генерации псевдослучайных чисел. В связи с этим Сэмплинг и Разбиение на множества могут генерировать другое разбиение. В обработчике Заполнение пропусков иначе работает метод «Заполнение случайными значениями». В Квантовании по-другому работает метод разбиения «Плитка» с типом обработки совпадающих наблюдений «Назначить случайно».
  3. Internet Explorer browser is not supported.
  4. Не поддерживаются 32-разрядные операционные системы семейства Windows. Minimum requirements: Windows 10, Windows Server 2019.
